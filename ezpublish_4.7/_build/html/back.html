  • Moteur de recherche intégré ou déporté (ezFind / Solr)
  • Workflow intégré
  • Système de gestion de droits / Rôles / Utilisateurs
  • Extensions

Principes généraux

Définitions

Classe Schéma d'un type de contenu
Objet Instance d'une classe de contenu, créée par un contributeur
Version Version publiée / archivée - Historique des modifications d'un objet
Noeud Positionnement(s) d'un objet dans l'arborescence
Template Gabarit de mise en forme d'une vue
Override Surchage d'un comportement par défaut du noyau ou extension

Contenus

  • Données
  • Structure
  • Contenus + Gabarits » Page web

Gabarits

  • Mise en forme des données : HTML / CSS / JS
  • Séparation des métiers :
    • Rédacteurs
    • Webdesigner
    • Développeurs
  • Réutilisation / Factorisation

Classes

  • Structure du contenu
  • Evolutive
  • Ne contient pas les données
  • Attributs
    • Ligne de texte
    • Bloc XML (contenu riche)
    • Image
    • Relations d'objets
    • ...

Noeuds

  • Contenu publié / masqué
  • Instance d'un objet
    • dans une version donnée
    • dans un emplacement d'arborescence donné
  • Notion de multi-publication

Versions

  • Un objet contient les versions
  • Un contenu peut avoir plusieurs versions
  • Une seule version par langue est publiée
  • Une nouvelle version est crée à chaque modification (brouillon)
  • Possibilité de revenir à une version précédente : mais attention création d'une nouvelle version

Modules et vues

Modules

  • Fonctions spécifiques indépendantes des noeuds de l'arborescence
  • Accessibles via une URL
  • Modules du noyau ou spécifiques
  • Exemples :
    • /user/login
    • /content/search

Vues

  • Mise en forme d'un contenu (noeud/objet) dans un conteneur
  • Exemples :
    • Vue complète du noeud (accessible via son URL d'arborescence)
    • Vue simplifiée en page d'accueil dans un bloc
    • Vue simplifiée dans un diaporama
    • Vue d'édition du contenu

Sections

Définition

  • Une section est affectée à un noeud et à sa sous arborescence
  • Application de droits d'utilisateurs spécifiques
  • Paramétrage (ex : surcharge de templates)
  • Exemples :
    • Standard
    • Media
    • Users

Gestion des utilisateurs

Utilisateur

  • Un utilisateur est un objet de contenu
  • Le noeud peut appartenir à plusieurs groupes (multi-publication)
  • Affectation de droits à des groupes d'utilisateurs
    • Lecture d'une sous-arborescence
    • Création de contenus (contributeurs)
    • ...

Droits

  • Action liée à un module / une vue
  • Paramétrage administrateur en back-office
  • Gestion par groupes rassemblant des permissions
  • Exemples :
    content/read Visualisation d'un contenu
    content/edit Edition d'un contenu (contribution)
    user/login Possibilité de se logguer sur le siteaccess

Droits / Limitations

  • Classes
  • Noeud
  • Classe parente
  • Section
  • Sous-arborescence
  • ...
  • Exemples :
    • content/read + article : accès en lecture aux objets de la classe "article"
    • user/login + edition : possibilité de se logguer sur le back-office
    • content/create + folder + noeud #123 : possibilité de création d'un dossier uniquement dans la sous-arborescence du noeud #123

Rôles

  • Association de plusieurs droits à un "profil" d'utilisateur / groupe(s)
  • Exemples :
    • Anonyme : droit de lecture sur les sections Standard et Media
    • Contributeur actualités : droit de création/modification sur des contenus d'une sous-arborescence Actualités
    • Contributeur boutique : droit de création/modification sur des produits/catégories

Groupe d'utilisateurs

  • Un groupe contient une liste d'utilisateurs
  • On lui affecte un ou plusieurs rôles
  • Possibilité de groupes / sous-groupes / ...
  • Système d'héritage des droits entre les groupes / sous-groupes

Workflow

  • Déclenchement d'événements via le navigateur ou déléguée à un cron
  • Déclencheurs
    • content/publish
    • content/delete
    • shop/addtobasket
  • Evénements
    • Before : avant l'action content/publish par ex
    • After : après l'action content/delete par ex

Formulaire

  • Contenu classique
  • Option "Collecteur d'information" sur les attributs
  • Soumissions du fomulaire enregistrées
  • Paramétrage de mails client/admin

Datatype

  • Type d'attribut ayant un ensemble de règles de gestion
    • Ligne de texte (= champ texte limité à X caractères)
    • Relation d'objet (limitation à des classes de contenu)
    • ...
  • Possibilité de créer un datatype spécifique
    • Format d'enregistrement en base de données
    • Contrôles de formulaires en back-office
    • Création de toutes les vues nécessaires FO/BO car n'existe pas dans le design standard
